# Proposal: Container Strategy for Safe

**Target version:** v0.3
**Related TBDs:** TBD-04 (Library subset boundary and curation policy)
**Related decisions:** D16 (no generics), D17 (access types with ownership), D18 (no tagged types), D27 (Silver-by-construction)

## Problem

Safe excludes all `Ada.Containers.*` packages because they rely on generics (D16),
tagged types for iteration (D18), controlled types for memory management, and
exceptions for constraint violations.  Without containers or a bounded string
buffer, developers must build dynamic data structures from scratch using access
types and ownership -- technically possible but a serious adoption barrier.

## Background: SPARK's Formal Containers

SPARK provides a full container library (`SPARK.Containers.Formal.*`) with
bounded and unbounded variants of vectors, lists, maps, and sets.  Key design
points relevant to Safe:

- **Preconditions replace exceptions.**  Instead of raising `Constraint_Error`,
  `Element` has `Pre => Has_Element(Container, Position)`.  GNATprove must prove
  the precondition holds at every call site.  This model transfers directly to
  Safe's D27 rules.

- **Bounded containers use discriminants for capacity** -- a record with a
  `Capacity` discriminant and a fixed-size internal array.  No heap allocation.

- **Every container is a generic package.**  There is no non-generic version.
  This is the fundamental incompatibility with Safe.

- **Implementations are not proven.**  The `.ads` specs have full SPARK
  contracts; the `.adb` bodies are not in SPARK.  GNATprove verifies correct
  *usage* of containers, not the container internals.

### Compatibility matrix

| SPARK container feature       | Safe restriction          | Compatible? |
|-------------------------------|---------------------------|-------------|
| Generic packages              | D16: no generics          | No          |
| Tagged types (internal)       | D18: no tagged types      | No          |
| Iterable aspect for `for..of` | Excluded for containers   | No          |
| Controlled types              | Excluded                  | No          |
| Precondition-based API        | Matches D27 model         | Yes         |
| Bounded discriminant-capacity | Just a record + array     | Yes         |
| Cursor-as-integer API         | If no tagged/access cursor| Yes         |
| Heap allocation (unbounded)   | D17 ownership types       | Potentially |

## Proposed Strategy

Three phases, each self-contained and shippable:

### Phase 1: Monomorphic built-in containers (v0.2--v0.3)

Ship a small set of non-generic, type-specific container packages in the
retained library surface:

```
Safe.Integer_Vectors         -- bounded vector of Integer
Safe.Natural_Vectors         -- bounded vector of Natural
Safe.Character_Buffers       -- bounded string buffer
Safe.Integer_Ordered_Sets    -- bounded ordered set of Integer
```

Each is what you get after instantiating a SPARK formal container with a
specific element type.  The implementations are Safe-legal: arrays with
discriminant capacity, precondition-guarded APIs, no generics, no exceptions,
no tagged types.

**Advantages:**
- Complies with all Safe restrictions as written.
- Precondition-based error handling model from SPARK transfers directly.
- Covers the most common use cases (integer collections, string buffers).

**Disadvantages:**
- One package per element type -- does not scale to user-defined types.
- Tedious to write and maintain (though each is straightforward).

### Phase 2: Emitter-based container instantiation (v0.3--v0.4)

The spec already notes that D16 applies to Safe source, not emitted Ada.  The
compiler uses `Ada.Unchecked_Deallocation` generics in emitted code.  Extend
this principle to containers:

```ada
-- Safe source syntax (strawman)
type Readings is bounded_vector of Sample capacity 256;
```

The emitter generates:

```ada
-- Emitted Ada
package Readings_Pkg is new SPARK.Containers.Formal.Vectors
  (Index_Type   => Positive,
   Element_Type => Sample,
   Capacity     => 256);
subtype Readings is Readings_Pkg.Vector;
```

The Safe programmer writes monomorphic types; the compiler maps them to proven
SPARK container instantiations in the output.

**Advantages:**
- Works for any element type, not just predefined ones.
- Leverages existing, contract-rich SPARK container infrastructure.
- Keeps generics out of Safe source (D16 preserved).

**Disadvantages:**
- Requires designing the Safe-side syntax and mapping rules.
- The Safe compiler must understand the SPARK container API to generate
  correct narrowing-point checks.
- Creates a dependency on SPARKlib in the emitted code.

### Phase 3: Restricted generics (v0.5+)

A monomorphic-specialization model (like Zig's `comptime` or Rust's
monomorphisation) where generic packages are expanded at compile time into
concrete types.  Each instantiation produces a distinct compilation unit with
no runtime polymorphism.

```ada
-- Possible Safe syntax
generic
   type Element_Type is private;
package Bounded_Vectors is
   type Vector (Capacity : Positive) is private;
   procedure Append (V : in out Vector; E : Element_Type)
      with Pre => Length(V) < V.Capacity;
   ...
end Bounded_Vectors;

-- Instantiation (monomorphised at compile time)
package Sensor_Vectors is new Bounded_Vectors (Sensor_Reading);
```

**Advantages:**
- Full generality -- users can write their own generic containers.
- Compatible with separate compilation (each instantiation is a unit).
- No runtime polymorphism, no vtables, no dynamic dispatch.

**Disadvantages:**
- Touches D16 directly -- requires revisiting the generics exclusion.
- Needs careful interaction analysis with ownership, channels, D27 proofs.
- Highest design complexity of the three phases.

## Strings

Safe can handle strings as Ada's built-in `String` type (which is
`array (Positive range <>) of Character`) with slicing.  For bounded mutable
strings, Phase 1's `Safe.Character_Buffers` provides a record with a
fixed-capacity character array and a length counter, with precondition-guarded
`Append`, `Slice`, `Length`, `Element`, and basic search operations.

This replaces `Ada.Strings.Bounded.Generic_Bounded_Length` (which is generic)
with a monomorphic, Safe-legal equivalent.
