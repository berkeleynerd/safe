Ada 2022 Language Reference Manual

The documents on this page describe the 2022 revision to the Ada Standard, usually known as Ada 2022. The revisions were produced by the ISO/IEC JTC 1/SC 22/WG 9 Ada Rapporteur Group (ARG), and have been published as the International Standard ISO/IEC 8652:2023(E). This document is often known as the Ada Reference Manual. Related documents are also found on this page. These documents are not an official publication or work product of the ARG, but rather are provided as a service to the Ada community.
Older versions of the Ada Reference Manual are also available. The consolidated Ada 2012 standard (including TC1) is available here. The original Ada 2012 standard is available here. The Ada 2005 consolidated standard is available here.
Send editorial comments on the documents to agent@ada-auth.org. Editorial comments are those that do not change the meaning of the text, such a spelling errors, doubled words, etc. [Note: Comments on paragraph and page layout should only be made for the RM-Final.pdf; other versions are derived from that primary version and cannot be changed separately.]
Substantive comments should be submitted by one of the methods outlined in the Introduction of the standard.

  • Annotated Ada 2022 Language Reference Manual (AARM)
    Intended primarily for compiler writers, conformance test writers, and others who wish to study the fine details, the AARM contains all of the text and various annotations in the Ada 2022 LRM. This 2022 edition includes the all of the changes in the 4th Edition of the Ada Reference Manual as well as new and revised annotations.

  • Ada 2022 Overview
    The Ada 2022 Overview provides an overview of new Ada 2022 features, examples of their use, compatibility with Ada 95, 2005, and 2012, and more. It was written by Jeff Cousins. The Overview is available on-line in HTML and PDF formats.
  • Ada Reference Manual Source Files
    The source files used to create the Ada Reference Manual are available here. These should be of interest only to editors of the ARM. These files are used as input to the formatting tool (available below), and are in a Scribe-like language.
