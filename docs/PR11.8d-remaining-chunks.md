## Remaining Chunk

### Chunk 5: Borrow Semantics and Parameter Modes

**Status**: Depends on PR11.8e.

This is the remaining substantive follow-up for the original PR11.8d plan:
wire the roadmap’s borrow-on-pass semantics onto the PR11.8d string and
array types once PR11.8e settles the source-level parameter/reference model.

Planned scope after PR11.8e:

- `in` parameters borrow without deep copy
- mutable-borrow semantics follow the PR11.8e parameter model
- assignment remains copy-by-default
- return remains ownership transfer
