## Former Remaining Chunk

### Chunk 5: Borrow Semantics and Parameter Modes

**Status**: Done (PR11.8e).

This chunk closed when PR11.8e reset the parameter/reference surface to
ordinary immutable borrows plus `mut` mutable borrows. That PR carried the
remaining PR11.8d borrow-on-pass work for strings and growable arrays under
the new model.

Delivered scope in PR11.8e:

- ordinary parameters borrow without deep copy
- `mut` parameters borrow mutably
- assignment remains copy-by-default
- return remains ownership transfer

## Deferred to Later Milestones

These items were discussed during PR11.8d design but are explicitly not
part of the PR11.8d completion line:

- **String/growable-array channel elements** — deferred to PR11.8g
  (Value-Type Channel Elements).
- **String discriminants** — strings are not discrete; still deferred.
- **Broader proof-fact growable-to-fixed narrowing** — direct guarded
  `values.length == N` support shipped in PR11.8d.1; wider MIR/proof-fact
  integration remains deferred.
- **MIR lint pass for Copy/Free completeness** — useful but not blocking;
  can be added as a standalone CI check later.
- **Valgrind/AddressSanitizer leak checking** — documented as a manual
  validation step, not a hard repo dependency.
